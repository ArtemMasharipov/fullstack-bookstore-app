# Screen Playbook

## Home

- Build clear premium hero with one dominant CTA.
- Add trust and differentiation blocks (why buy here).
- Surface curated rails: new, trending, editor picks.

## Books Listing

- Keep search/filter/sort visible and fast.
- Prioritize scannability: cover, title, author, price, stock, CTA.
- Support loading, empty, and no-results states explicitly.

## Book Details

- Keep pricing, availability, and add-to-cart highly visible.
- Add delivery/return trust cues near CTA.
- Add related/cross-sell block with clear relevance.

## Authors + Author Details

- Present author value, not only biography dump.
- Bridge author pages to purchasable books quickly.

## Cart

- Keep quantity edits frictionless and reversible.
- Keep order summary visible (sticky on long pages).
- Make next action explicit: checkout/login.

## Checkout

- Prefer multi-step clarity when form is non-trivial.
- Reduce required fields to minimum.
- Keep inline validation immediate and understandable.
- Keep final pay/place-order CTA persistent on mobile.

## Login/Register/Profile

- Keep auth forms short and recovery-oriented.
- Show clear error handling and continuation paths.
- Keep profile UI in same design system as storefront/admin.

## Orders

- Show status progression, totals, and support path.
- Enable reorder where relevant.

## Admin

- Prioritize actionable insights over decorative metrics.
- Standardize tables/forms/dialogs and bulk operations.
- Keep filters/search/sort stable and predictable.
